CodeForces 1146C - Tree Diameter

Start practicing graph. Solution The first method is the standard algorithm to find the diameter of the tree: Find the furthest distance from node 1 and then find the furthest distance from that node. This also works on this problem. We first use one query to find the furthest distance form node 1. Then use binary search to find which node is the furthest. The second way is what I think the coolest and concise way.

Codeforces 1187E - Tree Painting

换根有点意思的。 Solution We can calculate the point if we fix the starting node in $O(n)$ time. In order to get the final answer, we need to calculate all the points of all the starting nodes, which will give us a $O(n^2)$ solution, too slow. Actually, we only need to calculate the point once and we can easily get the points for other staring points: Suppose we have calculated the $point$ of some start vertex $u$ and we want to know the point for some other starting vertex $v$.

Codeforces 1248D - Segment Tree

这跟暴力又有什么区别呢？ Solution( based on the official editorial) Intersections can be found with sweep line approach (store the endpoints of each segment and its index, sort the array and iterate over it).We need to maintain a set of the endpoints of the open segments. When we add a segment, we iterate over the open segments ad find segments which end earlier than it. It looks like brute force,right? but we don’t need to worry about TLE since we will stop when the number of intersection is greater than $n-1$.
